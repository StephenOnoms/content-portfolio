Every open-weight model you'll ever need.

Lighthouse speaks the OpenAI chat completions format, so the SDK and code you already have keep working — just point them at a different base URL and pick a model from a catalog of thousands.

Read the quickstart → Switching models guide
python 
from openai import OpenAI

client = OpenAI(
    base_url="https://api.lighthouse.dev/v1",
    api_key="LIGHTHOUSE_API_KEY",
)

response = client.chat.completions.create(
    model="Qwen/Qwen2.5-32B-Instruct",
    messages=[{"role": "user", "content": "Explain cold starts in one sentence."}],
)

print(response.choices[0].message.content)

Frequently asked

Do I need to change my code to switch models?

No — change the model field in your request to any model name from the Lighthouse catalog. Everything else (messages, parameters, response format) stays the same. See the model-switching guide for a worked example.

Is there a rate limit?

Rate limits are tied to your plan's concurrency tier rather than a fixed requests-per-minute number — see the pricing page for details on what each tier supports.

Can I use this with LangChain?

Yes — configure LangChain's OpenAI-compatible chat model class with Lighthouse's base URL and your API key, then use any model name from the catalog. No additional packages required.

What happens if a model I'm using is updated or deprecated?

We pin model versions by default, so your requests keep working with the version you started with. When we add a newer version of a model, it appears under a new model name — your existing integration is unaffected until you choose to switch.
