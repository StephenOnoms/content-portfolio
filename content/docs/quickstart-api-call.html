Quickstart

Your First API Call

~5 min · Lighthouse Docs

This guide gets you from signup to a working response in about five minutes. By the end, you'll have an API key, you'll have made one request, and you'll know where to go next depending on what you're building.

In this guide

  • Step 1: Create an account and generate an API key
  • Step 2: Make your first request with curl
  • Step 3: Make the same request with the OpenAI Python SDK
  • Step 4: Understand the response, including usage data
  • Next steps: where to go from here

Step 1: Create an API Key

Sign up at lighthouse.dev and open your dashboard. Under API Keys, click Create new key and give it a name (e.g. dev-local) so you can identify it later if you create more.

Copy the key now — for security, it's only shown once. Store it as an environment variable rather than pasting it directly into code:

export LIGHTHOUSE_API_KEY="your-key-here"

Note

New accounts start on the free Developer tier automatically — no credit card required for this quickstart.

Step 2: Make Your First Request with curl

The Lighthouse API uses the same request format as OpenAI's chat completions endpoint. Here's a minimal request:

curl https://api.lighthouse.dev/v1/chat/completions \
  -H "Authorization: Bearer $LIGHTHOUSE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Llama-3-8B-Instruct",
    "messages": [
      {"role": "user", "content": "Say hello in one sentence."}
    ]
  }'

You should get a JSON response within a few seconds. If this is the first request to this model from your account, it may take slightly longer — see the note on cold starts below.

Step 3: Make the Same Request with the OpenAI Python SDK

Because the API is OpenAI-compatible, you can use the official OpenAI Python SDK by changing only the base_url and api_key:

pip install openai
from openai import OpenAI

client = OpenAI(
    base_url="https://api.lighthouse.dev/v1",
    api_key="your-key-here",  # or os.environ["LIGHTHOUSE_API_KEY"]
)

response = client.chat.completions.create(
    model="meta-llama/Llama-3-8B-Instruct",
    messages=[
        {"role": "user", "content": "Say hello in one sentence."}
    ],
)

print(response.choices[0].message.content)

Step 4: Understanding the Response

The response includes the generated message plus a usage object showing token counts:

{
  "id": "chatcmpl-...",
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "Hello! It's great to meet you."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 12,
    "completion_tokens": 9,
    "total_tokens": 21
  }
}

The usage object is worth logging from day one — even though the Developer and Growth plans are flat-rate, tracking token usage per model helps you spot when a feature's prompts are growing (e.g. as you add more context or conversation history), which is useful context if you ever need to debug latency or plan a tier change.

About cold starts

If a model hasn't been called recently by any Lighthouse user, the first request may take longer while it's loaded. Subsequent requests to the same model are fast. If your application is latency-sensitive for a specific model, consider sending an occasional "warm-up" request during low-traffic periods.

Next Steps

  • Try a different model by changing the model field — see the model-switching guide for a walkthrough and a list of popular models.
  • Enable streaming by adding "stream": true to your request if you're building a chat interface and want to display tokens as they arrive.
  • Check your usage dashboard to see request volume and concurrency — useful before deciding whether to move off the free tier.
